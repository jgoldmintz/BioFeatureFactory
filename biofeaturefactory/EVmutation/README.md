# EVmutation / PLMC Pipelines

## Overview

This directory contains two complementary pipelines:

1. **`evmutation-pipeline.py`** -- protein-level mutation effects for missense variants using amino acid MSAs
2. **`evmutation-pipeline.py`** -- codon-level mutation effects for synonymous variants using codon-aware MSAs

Both pipelines fit a **Potts (pairwise maximum-entropy) model** via plmc and score mutations as changes in the model Hamiltonian.

## Theoretical Background

### Potts Model

Both pipelines fit a maximum-entropy distribution over sequences:

$P(A_1, \ldots, A_L) = \frac{1}{Z} \exp\left(\sum_i h_i(A_i) + \sum_{i \lt j} J_{ij}(A_i, A_j)\right)$

- $h_i(A_i)$ -- site-specific field: how strongly position $i$ prefers character $A$ based on single-site frequencies in the MSA
- $J_{ij}(A_i, A_j)$ -- pairwise coupling: the residual statistical dependence between positions $i$ and $j$ after single-site effects are accounted for
- $Z$ -- partition function (normalisation constant)

### Mutation Effect ($\Delta H$)

For a single substitution at position $i$ (wildtype $a \to$ mutant $b$) against the wildtype background $\mathbf{s}$:

$\Delta H = \underbrace{\left[h_i(b) - h_i(a)\right]}_{\text{independent}} + \underbrace{\sum_{j \neq i} \left[J_{ij}(b,\, s_j) - J_{ij}(a,\, s_j)\right]}_{\text{epistatic contribution}}$

The **independent score** uses $h_i$ only (equivalent to `model.to_independent_model()`).
The **epistatic score** adds the full $J_{ij}$ sum over all other positions $j$ in the wildtype background.

**Sign convention**: $\Delta H > 0$ = tolerated/favored; $\Delta H < 0$ = deleterious.

## Pipeline 1: Protein EVmutation (`evmutation-pipeline.py`)

Scores **missense** and **stop** mutations using an amino acid MSA.

### Workflow

```
Protein MSA (A2M / FASTA)
        |
        v
  plmc (Potts inference)
        |
        v
  .model_params (binary)
        |
        v
  CouplingsModel + single_mutant_matrix()
        |
        v
  {GENE}.protein.tsv
```

### Input Requirements

- **Protein MSA**: A2M or FASTA format; jackhmmer / HHblits against UniRef90 recommended
- **ORF FASTA**: nucleotide ORF for mutation-to-amino-acid mapping
- **Mutations CSV**: list of nucleotide mutations

### Output: `{GENE}.protein.tsv`

| Column | Description |
|--------|-------------|
| `pkey` | `GENE-mutation` identifier |
| `nt_mutant` | Source nucleotide mutation (e.g., `G>A`) |
| `codon_position` | Codon position in ORF (1-based) |
| `wt_codon` / `mut_codon` | Wildtype and mutant codons |
| `mutant` | Amino acid substitution string (e.g., `V123A`) |
| `pos` | Amino acid position (1-based) |
| `wt` / `subs` | Wildtype / substituted amino acid |
| `mutation_class` | `MISSENSE`, `STOP_GAIN`, `STOP_LOSS`, `SYNONYMOUS`, `UNKNOWN` |
| `prediction_epistatic` | Full Potts model $\Delta H$ (primary score) |
| `prediction_independent` | Site-field-only $\Delta H$ (baseline) |
| `epistatic_contribution` | `epistatic - independent` |
| `site_entropy` | Shannon entropy at position (bits) |
| `mean_epistatic_at_pos` | Mean epistatic score across all substitutions at this position |
| `std_epistatic_at_pos` | Std of epistatic scores at this position |
| `z_score_epistatic` | Z-score relative to all substitutions at this position |
| `frequency` | Observed frequency of substitution in MSA |
| `column_conservation` | Max single-AA frequency at this position |
| `qc_flags` | See QC Flags section |

### Score Interpretation (Protein)

| Score range | Interpretation |
|-------------|----------------|
| > 0 | Tolerated / likely neutral |
| 0 to -2 | Mild effect |
| -2 to -5 | Moderate deleterious |
| < -5 | Strongly deleterious |

`prediction_epistatic` is the primary score when $N_{eff} \gg 6{,}500$ (for a 20-character alphabet with standard L2 regularisation). At lower $N_{eff}$, `prediction_independent` is more reliable.

### Usage

```bash
# Full pipeline (run plmc)
python evmutation-pipeline.py \
    --msa BRCA1.msa.a2m \
    --focus HUMAN_BRCA1 \
    --plmc-binary /usr/local/bin/plmc \
    --output BRCA1.evmutation.tsv

# Pre-computed parameters
python evmutation-pipeline.py \
    --model-params BRCA1.model_params \
    --output BRCA1.evmutation.tsv

# With nucleotide mutation mapping
python evmutation-pipeline.py \
    --msa BRCA1.msa.a2m \
    --focus HUMAN_BRCA1 \
    --plmc-binary /usr/local/bin/plmc \
    --map combined_BRCA1.csv \
    --output BRCA1.evmutation.tsv
```

## Pipeline 2: Codon EVmutation (`evmutation-pipeline.py`)

Scores **synonymous** mutations using a codon-aware MSA. This pipeline addresses the limitation that protein EVmutation assigns no score to synonymous variants. It trains a separate Potts model where each sequence position represents a **codon triplet** rather than an amino acid.

### Workflow

```
Codon MSA (triplet FASTA)
        |
        v
  encode_codon_msa()        <- bin/codon_encoding.py
  (3-nt triplets -> single chars)
        |
        v
  plmc (Potts inference, 64-char codon alphabet)
        |
        v
  .codon_model_params (binary)
        |
        v
  CouplingsModel + single_mutant_matrix()
  + col_to_seq remapping
        |
        v
  {GENE}.codon.tsv
```

### Codon Encoding

The codon MSA uses triplet codons as alignment tokens. Before plmc can process it, each codon is encoded as a single character. `bin/codon_encoding.py` maps all 64 sense codons to a 64-character alphabet using lexicographic ACGT ordering:

```
AAA -> 'A'   AAC -> 'B'   ...   TTT -> '@'   TTC -> '9'   gap (---) -> '-'
```

The encoded MSA has the same number of columns as the original codon MSA (one column per codon position), allowing plmc to learn $J_{ij}$ between codon positions.

### What "Coupling" Means in a Codon Model

In a **protein** EVmutation model, $J_{ij}$ reflects structural and functional co-evolution between amino acid positions -- residues in contact tend to co-vary to maintain fitness.

In a **codon** model, $J_{ij}$ captures statistical co-variation in **codon choice** between CDS positions across the MSA. The primary driver is **phylogenetic codon usage bias**: a species with high genomic GC content will tend to use C-ending synonymous codons (e.g., TTC, CAG) across many positions simultaneously, while a species with AT bias uses the complement (TTT, CAA). When position $i$ uses TTC in a sequence, position $j$ in that same sequence also tends to use a C-ending codon -- not because positions $i$ and $j$ are physically coupled, but because they share the same host organism's codon usage signature.

$J_{ij}$ is the maximum-entropy parameter required to reproduce the **observed pairwise codon frequencies** at positions $i$ and $j$ across the MSA, after the single-site frequencies ($h_i$) are already accounted for. It is nonzero only when the codon at $i$ and the codon at $j$ are chosen **non-independently** -- i.e., knowing the codon at position $i$ gives information about the codon at position $j$ beyond what the marginal frequencies predict.

Secondary drivers of codon coupling can include:
- **Translational co-regulation**: selection to maintain rare codon clustering patterns for co-translational folding
- **tRNA pool effects**: co-selection of translation speed between positions that influence the same folding event

### Input Requirements

- **Codon MSA** (Required): triplet FASTA generated by `../utils/codon_msa_pipeline.py`; each sequence is a concatenation of 3-nt codons; gaps as `---`
- **ORF FASTA** (Required): nucleotide CDS for the focus gene; used to determine wildtype codons at each position; must contain a sequence with ID `ORF` (or the first record is used)
- **Mutations CSV** (Required): list of nucleotide mutations to score
- **Codon model params** (Optional): pre-computed `.codon_model_params` binary; if absent and `--codon-msa` is given, plmc is run automatically

### Output: `{GENE}.codon.tsv`

| Column | Description |
|--------|-------------|
| `pkey` | `GENE-mutation` identifier |
| `nt_mutant` | Source nucleotide mutation (e.g., `T840C`) |
| `codon_position` | Codon position in ORF (1-based) |
| `wt_codon` / `mut_codon` | Wildtype and mutant codons |
| `mutation_class` | `SYNONYMOUS`, `STOP_GAIN`, or `STOP_LOSS` |
| `prediction_codon_independent` | Site-field-only $\Delta H$ (primary score; see below) |
| `prediction_codon_epistatic` | Full Potts model $\Delta H$ |
| `codon_epistatic_contribution` | $\Delta H_{epi} - \Delta H_{indep}$ = $\sum_{j \neq i} [J_{ij}(b, s_j) - J_{ij}(a, s_j)]$ |
| `codon_epistatic_concordance` | `CONCORDANT`, `DISCORDANT`, or `NEUTRAL` |
| `codon_frequency` | Observed frequency of mutant codon at this position in the MSA |
| `qc_flags` | See QC Flags section |

## Score Interpretation (Codon)

### `prediction_codon_independent`

$\Delta H_{indep} = h_i(b) - h_i(a)$

Reflects how strongly the MSA favors the mutant codon $b$ over the wildtype codon $a$ at position $i$, based on **single-site codon frequencies only**. A positive value means the mutant codon is more frequent at this position than the wildtype codon across the alignment. A negative value means it is rarer.

This is the **primary score** for synonymous mutations. It is driven by the cross-species distribution of codon usage at each CDS position and is robust to low $N_{eff}$ because it depends only on marginal frequencies, not pairwise statistics.

### `prediction_codon_epistatic`

$\Delta H_{epi} = \Delta H_{indep} + \sum_{j \neq i} \left[J_{ij}(b,\, s_j) - J_{ij}(a,\, s_j)\right]$

The full Potts model score. Adds the contribution of all pairwise couplings to the site-specific preference. In high-$N_{eff}$ models (deep, diverse MSAs), this captures whether the mutant codon fits the **broader codon usage context** of the focus sequence -- i.e., whether sequences that use the mutant codon at position $i$ also tend to use the wildtype codons present elsewhere in the focus sequence.

At low $N_{eff}$, the $J_{ij}$ matrix is heavily L2-regularised toward zero, making the epistatic score converge toward the independent score. Use `prediction_codon_independent` as the primary score in that regime.

**$N_{eff}$ guidance for codon models** (64-character alphabet):

| $N_{eff}$ | Reliability |
|-----------|-------------|
| $\geq 640 \times L$ | Epistatic score fully reliable |
| $100$ to $640 \times L$ | Epistatic score noisy; use independent score |
| $< 100 \times L$ | Epistatic scores regularisation-dominated; independent score only |

The threshold is higher than for protein models because the codon alphabet has 64 characters vs. 20 for amino acids, requiring more sequences to estimate pairwise statistics.

### `codon_epistatic_contribution`

$\Delta H_{contrib} = \Delta H_{epi} - \Delta H_{indep} = \sum_{j \neq i} \left[J_{ij}(b,\, s_j) - J_{ij}(a,\, s_j)\right]$

The marginal effect of pairwise couplings in isolation, evaluated against the wildtype codon background at all other positions. Answers: **given the wildtype codon at every other CDS position, do the pairwise coupling terms favor or oppose this substitution?**

- **Negative**: the mutant codon is disfavored by coupling to the wildtype background -- sequences that use the mutant codon at position $i$ tend to have different codon preferences at other positions than the wildtype sequence
- **Positive**: coupling partners partially favor the mutant codon -- the wildtype sequence happens to share codon usage patterns with sequences that also favor the mutant codon
- **Near zero**: pairwise couplings are neutral with respect to this substitution

### `codon_epistatic_concordance`

Classifies whether the epistatic and independent scores agree, accounting for the noise floor at each position:

```
threshold = 0.5 x std(contributions across all substitutions at position i)

CONCORDANT  -- |contrib| > threshold  AND  sign(epistatic) == sign(independent)
DISCORDANT  -- |contrib| > threshold  AND  sign(epistatic) != sign(independent)
NEUTRAL     -- |contrib| <= threshold  OR  std == 0  OR  contrib is NaN
```

- **CONCORDANT**: pairwise couplings reinforce the site-specific preference; the epistatic score adds information in the same direction
- **DISCORDANT**: couplings oppose the site-specific preference; the two scores disagree on direction; inspect $N_{eff}$ before interpreting
- **NEUTRAL**: the epistatic contribution is within the noise floor; the epistatic score adds no reliable information beyond the independent score

In low-$N_{eff}$ models, nearly all positions will be CONCORDANT or NEUTRAL because regularisation drives $J_{ij} \approx 0$. DISCORDANT cases become meaningful only at high $N_{eff}$.

### `codon_frequency`

The observed frequency of the mutant codon at this position in the MSA (`model.fi`). A direct count from the alignment, independent of the Potts model. Use this as a sanity check: `prediction_codon_independent` and `codon_frequency` should agree in direction -- a high-frequency codon should have a positive independent score.

### Example Interpretation

From SMN2 ($N_{eff}=113$, $L=294$ -- low $N_{eff}$ regime; primary score is `prediction_codon_independent`):

| Mutation | indep | epi | contrib | concordance | freq | Notes |
|----------|-------|-----|---------|-------------|------|-------|
| F280F (TTT->TTC) | +2.09 | +1.50 | -0.59 | CONCORDANT | 0.387 | TTC favored; 39% of MSA use TTC; couplings slightly oppose but agree in sign |
| S292S (TCC->TCG) | -8.03 | -6.75 | +1.28 | CONCORDANT | 0.000 | TCG never observed; strongly disfavored; couplings partially rescue but both remain negative |
| Q27Q (CAG->CAA) | -2.85 | -5.46 | -2.61 | CONCORDANT | 0.052 | Site penalizes CAA; couplings amplify the deleterious signal |

For a well-characterized gene with a deep MSA (e.g., TP53, F9), DISCORDANT cases would indicate genuine epistatic conflict between local codon preference and genome-wide codon usage context.

## Combined Pipeline: Inputs, Outputs, and Structure

`evmutation-pipeline.py` runs both models in a single process and writes two output tables per gene.

### Output Structure

```
<output>/
  <GENE>/
    EVmutation/
      <GENE>.protein.tsv    <- missense, unknown
      <GENE>.codon.tsv      <- synonymous, stop gain/loss
```

### Usage

```bash
# Single gene, pre-built params
python evmutation-pipeline.py \
    --fasta SMN2.fasta \
    --mutations SMN2.csv \
    --model-params SMN2.model_params \
    --codon-model-params SMN2.codon_model_params \
    --output results/

# Single gene, build params from MSAs
python evmutation-pipeline.py \
    --fasta SMN2.fasta \
    --mutations SMN2.csv \
    --msa SMN2.msa.a2m \
    --codon-msa SMN2.codon.msa.fasta \
    --plmc-binary /usr/local/bin/plmc \
    --output results/

# Multi-gene directory mode
python evmutation-pipeline.py \
    --fasta /data/fastas/ \
    --mutations /data/mutations/ \
    --msa /data/msas/ \
    --codon-msa /data/codon_msas/ \
    --plmc-binary /usr/local/bin/plmc \
    --output results/

# Codon only (no protein model)
python evmutation-pipeline.py \
    --fasta SMN2.fasta \
    --mutations SMN2.csv \
    --codon-model-params SMN2.codon_model_params \
    --output results/
```

### Parameters

#### Required
| Parameter | Description |
|-----------|-------------|
| `--fasta` | ORF FASTA file or directory of per-gene FASTA files |
| `--mutations` | Mutations CSV file or directory of per-gene CSVs |

#### Model Parameters (file or directory; at least one required for scoring)
| Parameter | Description |
|-----------|-------------|
| `--model-params` | Protein model params file or directory containing `{GENE}.model_params` |
| `--codon-model-params` | Codon model params file or directory containing `{GENE}.codon_model_params` |

#### MSA / plmc (trigger plmc if params absent)
| Parameter | Default | Description |
|-----------|---------|-------------|
| `--msa` | -- | Protein MSA file or directory |
| `--focus` | gene name | Focus sequence ID in protein MSA |
| `--codon-msa` | -- | Codon MSA file or directory |
| `--codon-focus` | `ORF` | Focus sequence ID in codon MSA |
| `--plmc-binary` | -- | Path to plmc executable (required when running plmc) |
| `--alphabet` | `-ACDEFGHIKLMNPQRSTVWY` | Protein alphabet |
| `--lambda-e` | 16.2 | $J_{ij}$ regularisation strength |
| `--lambda-h` | 0.01 | $h_i$ regularisation strength |
| `--skip-plmc` | False | Skip plmc inference (encoding still runs for codon MSAs) |

#### Other
| Parameter | Description |
|-----------|-------------|
| `--gene` | Gene name override (single-gene mode only) |
| `--validation-log` | Validation log from `exon_aware_mapping` for mutation filtering |
| `--output`, `-o` | Output directory (default: current directory) |
| `--quiet`, `-q` | Suppress verbose output |

### Model Parameter Resolution

For both `--model-params` and `--codon-model-params`:
1. If a **file** is given -- used directly
2. If a **directory** is given -- looks for `{GENE}.model_params` / `{GENE}.codon_model_params` inside it
3. If omitted and the corresponding `--msa` / `--codon-msa` is given -- params file is placed in the same directory as the MSA, then plmc is run if it does not yet exist
4. If both omitted -- that model is skipped

This allows MSAs and model params to reside in different directories (e.g., shared params cache vs. per-project MSA storage).

## MSA Quality Guidelines

### Protein MSA

| Metric | Minimum | Optimal |
|--------|---------|---------|
| $N_{eff}$ | >= 100 | >= 6,500 |
| Sequences | >= 100 | 1,000 - 10,000 |
| Max gaps/sequence | 40% | 30% |
| Max gaps/column | 60% | 50% |

### Codon MSA

| Metric | Minimum | Optimal |
|--------|---------|---------|
| $N_{eff}$ | >= 100 | $\geq 640 \times L$ |
| Sequences | >= 50 | 200 - 1,000 |
| Max gaps/sequence | 40% | 30% |
| Species diversity | Vertebrates | Multi-kingdom |

The codon MSA requires higher $N_{eff}$ than the protein MSA because the 64-character alphabet has many more free parameters in $J_{ij}$. With fewer than ~50 sequences, the epistatic terms will be entirely regularisation noise. `prediction_codon_independent` remains meaningful at any depth >= 10 sequences.

Generate a codon MSA using:

```bash
python ../utils/codon_msa_pipeline.py \
    --focus-protein SMN2_human.faa \
    --focus-nt-fasta SMN2.orf.fasta \
    --assembly-dir /path/to/refseq_assemblies/ \
    --max-hits 500 \
    --output SMN2.codon.msa.fasta \
    --manifest SMN2.codon.manifest.tsv
```

## Limitations

1. **Protein pipeline**: operates on amino acid sequences only; synonymous mutations receive no score
2. **Codon epistatic score**: `prediction_codon_epistatic` requires high $N_{eff}$ ($\geq 640 \times L$) to be reliable; `prediction_codon_independent` is reliable at any depth
3. **Single mutations only**: neither pipeline handles multi-nucleotide or compound variants
4. **Coverage gaps**: positions absent from the MSA receive no score (`NOT_IN_MODEL` / `SYNONYMOUS_NOT_IN_CODON_MODEL`)
5. **Codon coupling interpretation**: $J_{ij}$ in a codon model reflects codon usage co-variation (primarily phylogenetic GC/AT bias), not structural contacts -- do not interpret `codon_epistatic_contribution` as a physical interaction between CDS positions
