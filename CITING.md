# How to Cite BioFeatureFactory

BioFeatureFactory is open-source software, but **citation is required** in any
scientific, academic, clinical, or commercial work that uses it for analyses,
derivative tools, or services.

## Software citation

If you use BioFeatureFactory in a pipeline, analysis, web service, or product,
please cite it as:

> Goldmintz J.  
> **BioFeatureFactory: an open framework for multi-modal variant feature engineering.**  
> Software, 2026.  
> Available at: https://github.com/jgoldmintz/biofeaturefactory

When a formal publication is available, please prefer that citation; it will be
listed in this file and in the `CITATION.cff` metadata.

## Scope of citation

Citation is required when:

- You use BioFeatureFactory to generate features or scores for variants.
- You integrate BioFeatureFactory modules into an internal or external pipeline.
- You build a web service, API, or product that calls BioFeatureFactory.
- You publish results, benchmarks, or figures that were produced using
  BioFeatureFactory outputs.

This applies to:

- Academic and non-academic publications
- Preprints and technical reports
- Regulatory submissions
- Commercial and clinical reports
